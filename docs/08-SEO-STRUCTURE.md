# Planet Motors — SEO Structure

> **Last Updated:** 2026-04-30 (post-launch sprint)
> **Authoritative source:** `public/llms.txt` for AI agents, `app/sitemap.ts` for search engines.

## Overview

End-to-end SEO + AI-discovery implementation. Every page is server-rendered
with structured metadata, JSON-LD schemas, and sitemap entries that
auto-update as inventory changes. Search engines crawl naturally; we
accelerate discovery with IndexNow (Bing/Yandex) on every inventory
mutation.

// app/cars/[make]/page.tsx
import { parseCategorySlug } from '@/lib/seo/category-slug-parser'

export async function generateMetadata({ params }) {
  const { make } = await params
  const filter = parseCategorySlug(make)
  if (!filter) return { title: 'Page not found | Planet Motors' }
  return {
    title: filter.metaTitle,
    description: filter.metaDescription,
    alternates: { canonical: filter.canonicalPath },
  }
}
```

- `OrganizationJsonLd` — `@type: AutoDealer`, address, hours, phone
- `LocalBusinessJsonLd` — local SEO with aggregate rating

<VehicleJsonLd vehicle={vehicleDetail} />
<VehicleTrustJsonLd vehicle={vehicleDetail} />
```

`VehicleTrustJsonLd` adds machine-readable trust signals for AI agents:
210-point inspection status, 10-day return window, Aviloo battery
report URL when present, and accident-free policy declaration.

### Category Page Schema

Every `/cars/<slug>` page emits two JSON-LD blocks via
`<CategoryJsonLd>`:

1. `ItemList` listing every visible vehicle (position, name, image,
   price, availability) so AI agents see the full match set without
   executing JS.
2. `BreadcrumbList` for `Home → Inventory → <Category>`.

## 3. Sitemap Configuration

`app/sitemap.ts` aggregates four sub-sitemaps from
`lib/sitemap-builders.ts`:

| Sub-sitemap | Source | Priority | Change Frequency |
|---|---|---|---|
| Static pages | hardcoded list | 0.3 – 1.0 | yearly – daily |
| Inventory filters | hardcoded list | 0.75 – 0.9 | daily |
| Category landing pages | `enumerateCategorySlugs()` | 0.85 – 0.9 | hourly |
| Vehicle Detail Pages | live Supabase query | 0.75 | daily |
| Blog posts | `blogPosts` data | 0.6 | weekly |

### Static Pages (highlights)

| Page | Priority | Frequency |
|---|---|---|
| Homepage | 1.0 | daily |
| `/inventory` | 0.95 | hourly |
| `/financing` | 0.9 | weekly |
| `/electric-vehicles` | 0.95 | daily |
| `/about`, `/contact` | 0.8 | monthly |
| `/privacy`, `/terms` | 0.3 | yearly |

### Category Pages — `/cars/<slug>` (~150 URLs)

Auto-generated from `enumerateCategorySlugs()` so the sitemap and
the router never drift. Slugs are pre-validated by the parser; any
URL in the sitemap is guaranteed to render a real page.

```
Make-only:           /cars/tesla, /cars/porsche, /cars/genesis, ...
Body style:          /cars/sedan, /cars/suv, /cars/coupe, ...
Fuel type:           /cars/electric, /cars/hybrid, /cars/plug-in-hybrid
Premium tags:        /cars/luxury-evs, /cars/luxury-suvs, /cars/accident-free
Combos:              /cars/electric-suv, /cars/hybrid-sedan
Price bands:         /cars/under-30k, /cars/under-50k, /cars/under-100k
City crosses:        /cars/electric-in-toronto, /cars/tesla-in-richmond-hill
```

Cities served: `toronto`, `richmond-hill`, `markham`, `vaughan`,
`mississauga`, `brampton`, `oakville`, `hamilton`, `burlington`,
`niagara-falls`, `st-catharines`, `ontario`.

### Vehicle Detail Pages (live)

```typescript
async function buildVehiclesSitemap(baseUrl, currentDate) {
  const vehicles = await supabase
    .from('vehicles')
    .select('id, updated_at, primary_image_url, image_urls')
    .or(buildPublicStatusFilter())  // public = available + reserved + sold ≤ 7 days
    .order('updated_at', { ascending: false })
    .limit(10_000)

  return vehicles.map(v => ({
    url: `${baseUrl}/vehicles/${v.id}`,
    lastModified: v.updated_at,
    changeFrequency: 'daily',
    priority: 0.75,
    images: buildVehicleImages(v),  // up to 5 per vehicle
  }))
}
```

`<image:image>` tags feed Google Images so vehicle photos rank
independently. Soft-deleted (sold > 7 days) vehicles drop off
automatically.

### Image Sitemap

VDPs include their `primary_image_url` plus up to 4 supplementary
photos per sitemap entry. See `buildVehicleImages` in
`lib/sitemap-builders.ts`.

## 4. Robots.txt — AI Bot Policy

`app/robots.ts` ships a split policy:

### Allowed (recommendation crawlers)

These are on-demand search agents that cite sources only when they
can fetch them. We let them in so we get cited:

- `OAI-SearchBot` (ChatGPT Search)
- `ChatGPT-User` (ChatGPT browsing)
- `PerplexityBot`
- `ClaudeBot`

### Blocked (training crawlers)

Bulk scrapers that vacuum content for model training without
compensation. Explicitly disallowed:

- `GPTBot`, `Google-Extended`, `CCBot`
- `anthropic-ai`, `Claude-Web`
- `Bytespider`, `Applebot-Extended`, `cohere-ai`

### Standard Disallows (apply to all agents)

- `/api/`, `/admin/`, `/auth/*`, `/account/`, `/checkout/`, `/studio`

### Aggressive crawlers blocked

- `AhrefsBot`, `SemrushBot` (high request volume, no traffic ROI)

## 5. `public/llms.txt` (AI Discovery)

Markdown brief that AI agents fetch first to get the executive summary
of the business. Sized < 4 KB so it ingests in a single context window.

Sections:
1. **Core differentiators** — accident-free policy, Aviloo, no-fee pricing
2. **Key resources** — links to inventory, about, financing, EVs, Aviloo
3. **URL grammar for deep-linking** — Carvana-style pattern guide so AI
   agents can construct deep links to filtered inventory we didn't
   pre-generate (e.g., `/cars/<make>-<model>`)
4. **Geographic service area** — GTA primary, Canada-wide delivery
5. **Authoritative source** — declared canonical when scraped data
   conflicts

## 6. IndexNow Integration (Bing + Yandex)

Notifies Bing and Yandex within seconds whenever inventory changes.
Google does **not** support IndexNow; it discovers on its own crawl
schedule.

```typescript
// lib/seo/indexnow.ts
import { pingVehicleChange } from '@/lib/seo/indexnow'

// Called from admin POST/PUT/DELETE handlers:
await pingVehicleChange(vehicleId)
// Pings: /vehicles/<id>, /inventory  (batched)
```

Wired into:
- `app/api/v1/admin/vehicles/route.ts` (POST)
- `app/api/v1/admin/vehicles/[id]/route.ts` (PUT, DELETE)
- HomeNet sync cron (every 15 min, when net changes detected)

The API key is published at `/<key>.txt` per the IndexNow spec.

## 7. Search Engine Submission — Manual Setup (One-Time)

Sitemaps are automatically discovered via the `Sitemap:` directive in
`robots.txt`. But for fastest initial indexing, submit explicitly:

### Google Search Console

1. Add property at `https://search.google.com/search-console`
2. Verify domain ownership (DNS TXT record or HTML file)
3. **Sitemaps** → submit `https://www.planetmotors.ca/sitemap.xml`
4. **URL Inspection** → "Request Indexing" for top-priority pages
   (homepage, `/inventory`, `/electric-vehicles`, top 5 category
   pages) — accelerates first crawl from days → hours
5. Set up **Performance** alerts and **Coverage** error notifications

### Bing Webmaster Tools

1. Add site at `https://www.bing.com/webmasters`
2. Verify ownership
3. Submit `https://www.planetmotors.ca/sitemap.xml`
4. Already handled by IndexNow for ongoing changes

### Yandex Webmaster

1. Optional — only if you target Russian-speaking buyers
2. IndexNow handles ongoing notifications

| Engine | New URL discovery | Inventory change | First page-1 ranking |
|---|---|---|---|
| **Google** | 1–7 days from sitemap | 1–7 days (no IndexNow support) | 2–8 weeks |
| **Bing** | < 1 hour via IndexNow | < 1 hour via IndexNow | 1–4 weeks |
| **Yandex** | < 1 hour via IndexNow | < 1 hour via IndexNow | 2–8 weeks |
| **DuckDuckGo** | Mirrors Bing | Mirrors Bing | Mirrors Bing |
| **AI agents** (Perplexity, ChatGPT, Claude) | < 24 hours when llms.txt present | live (read on each query) | N/A — cited as a source |
